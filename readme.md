# Порівняння алгоритмів сортування

## Мета роботи
Порівняти ефективність трьох алгоритмів сортування:
- Insertion Sort
- Merge Sort
- Timsort (вбудований sorted у Python)

та емпірично перевірити їх теоретичну складність.

---

## Методика дослідження

Для вимірювання часу виконання використано модуль `timeit`.

Тести проведено на масивах різного розміру:
- 100 елементів
- 1000 елементів
- 5000 елементів

Кожен алгоритм виконувався кілька разів для отримання більш точних результатів.

---

## Результати експерименту

| Розмір масиву | Insertion Sort | Merge Sort | Timsort |
|---------------|----------------|------------|---------|
| 100           | 0.000278 s     | 0.000209 s | 0.000014 s |
| 1000          | 0.033729 s     | 0.002775 s | 0.000197 s |
| 5000          | 0.930648 s     | 0.017532 s | 0.001222 s |

---

## Аналіз результатів

### Insertion Sort
- Складність: O(n²)
- Різко погіршує продуктивність зі збільшенням розміру масиву
- Підходить лише для малих або майже відсортованих даних

### Merge Sort
- Складність: O(n log n)
- Має стабільну продуктивність
- Ефективний для великих наборів даних

### Timsort (sorted)
- Складність: O(n log n)
- Найшвидший у всіх тестах
- Поєднує сортування вставками та сортування злиттям
- Оптимізований для реальних даних

---

## Висновки

1. Емпіричні дані підтверджують теоретичні оцінки складності алгоритмів.
2. Insertion Sort є найменш ефективним на великих масивах.
3. Merge Sort демонструє стабільну ефективність O(n log n).
4. Timsort є найшвидшим завдяки гібридній структурі.
5. Вбудовані алгоритми Python є оптимальним вибором для практичного використання.
